## OSI 7계층이란??

![img](https://velog.velcdn.com/images/pppp0722/post/c3a64a22-e99b-414a-a228-8302ec874542/image.png)

OSI (Open Systems Interconnection) 모델
: 총 7개의 계층으로 이루어진 네트워크 아키텍처 모델
- 7개의 계층은 각각 특정 기능과 역할을 수행
- 네트워크 통신을 추상화
- 계층별로 구분함으로써 효율적인 설계와 유지보수 가능

1. 물리 계층 (Physical Layer)
   : 전송 매체와 관련된 하드웨어적인 부분 (케이블, 허브 등)

2. 데이터 링크 계층 (Data Link Layer)
   : 물리 계층 상에서의 오류 검출과 흐름 제어 담당 
   - 프레임의 동기화와 주소 결정을 수행
   - MAC (Media Access Control)주소와 같은 주소 사용

3. 네트워크 계층 (Network Layer)
   : 여러 개의 라우팅 경로 중에서 최적의 경로를 선택하여 패킷을 전달하는 역할
   - IP 주소를 사용하여 목적지 식별

4. 전송 계층 (Transport Layer)
   : 데이터의 신뢰성 있는 전송 담당
   - 흐름 제어와 오류 복구 수행
   - TCP (Transmission Control Protocol)
   - UDP (User Datagram Protocol)

5. 세션 계층 (Session Layer)
   : 양 끝단 간의 대화, 세션 설정 및 종료 등 관리
   -> 데이터 교환 조절

6. 표현 계층 (Presentation Layer)
   : 데이터의 인코딩, 압축, 암호화 등의 포맷 정의 및 변환

7. 응용 계층 (Application Layer)
   : 최종 사용자가 네트워크 서비스에 접근할 수 있도록 인터페이스 제공
   - 일반적인 응용프로그램과 네트워크 서비스에 접근하는 데 필요한 프로토콜 포함

### 내가 이걸... 알면 뭐가 좋지 ??

MAC 주소, IP 주소, 포트 번호는 네트워크 통신에서 중요한 역할을 하는 식별자

1. 네트워크 디버깅 및 문제 해결
   : MAC 주소와 IP 주소를 통해 네트워크 통신의 원인을 신속하게 식별 가능
   - 특정 장치 또는 서버 간의 통신 문제가 발생했을 때, 해당 디바이스의 MAC 주소와 IP 주소를 확인하여 문제 해결

2. 보안 강화
   : 특정 디바이스의 MAC 주소를 사용하여 네트워크에 접근을 허용 또는 거부하는 등의 보안 정책을 구현 가능
   - IP 주소와 포트 번호를 통해 트래픽 모니터링
   - 불법적인 액세스 또는 공격을 식별하는 데 도움

3. 네트워크 성능 최적화
   : 특정 서버 또는 디바이스의 트래픽을 추적, 포트 번호를 통해 어떤 종류의 서비스가 사용되고 있는지 파악 -> 네트워크 성능을 최적화
   - 병목 현상 예방, 성능 개선을 위한 조치

4. 프로그램 개발 및 통합
   : 특정 서비스나 어플리케이션을 개발할 때, IP 주소와 포트 번호를 사용하여 서버와의 통신을 설정
   - MAC 주소를 이용하여 특정 디바이스와의 통신 구현
   - 네트워크 기반의 기능 통합

5. 로그 및 감사 추적
   : 특정 디바이스의 MAC 주소와 IP 주소는 로깅 및 감사 추적 시 중요한 정보로 활용
   - 특정 이벤트 또는 트랜잭션을 추적하기 위해 정보 사용

MAC 주소(하드웨어 주소)
네트워크 장치, 특히 NIC의 고유 식별자

NIC가 LAN카드?
이 용어는 대부분 상호교체해서 사용, 모두 네트워크 장치를 가리키므로 맞는 말

Mac 주소는 물리적인 계층에서 작동
네트워크에서 서로 다른 장치들을 식별 가능

48비트 길이, 16진수로 표현
OUI : MAC주소의 첫 24비트, 제조사를 식별

NIC Identifier : 나머지 24비트, 해당 제조사에서 할당한 고유한 NIC 식별

그런데... 
"MAC 주소는 네트워크 계층에서의 IP 주소와는 다르게 물리적인 주소이기 때문에 변경이 어려우며, 일반적으로 네트워크 장치가 제조사에서 할당한 값을 그대로 사용"?

### 그렇다면 MAC주소는 변경이 불가능하다는걸까?

일반적으로 MAC 주소는 하드웨어 수준에서 고유한 값이기 때문에 물리적으로 변경이 불가능

이 주소는 네트워크 인터페이스 카드(NIC)에 하드웨어적으로 프로그래밍되어 제조사에 의해 할당
일반적인 운영에서 사용자 혹은 소프트웨어에 의해 직접 변경X

그러나 일부 운영 체제나 네트워크 장치에서는 소프트웨어적으로 MAC 주소를 변경하는 기능을 제공
특수한 경우나 특정 용도로 사용될 때 유용
예) 가상화 환경에서 가상 머신의 MAC 주소를 변경하는 등의 상황에서 활용
(네트워크 충돌 회피, 보안 및 익명성 강화, 네트워크 정책 대응, 가상머신 클론 및 마이그레이션)

MAC 주소를 변경하는 것은 네트워크 관리 정책이나 보안 정책에 따라 다를 수 O
사용자가 임의로 변경하는 것은 권장 X
-> 네트워크 트래픽의 추적 및 관리, 보안 정책 준수 등을 고려하기 때문

IP 주소
호스트(Host)를 식별하는 주소
IP(인터넷 프로토콜) 주소는 컴퓨터나 네트워크 장치를 구별하고 통신하기 위해 사용

호스트는 일반적으로 컴퓨터, 서버, 라우터, 또는 다른 네트워크 장치를 의미

IP 주소는 고유하게 할당되며, 전 세계적으로 네트워크 상에서 특정 호스트를 식별

IPv4 주소는 일반적으로 32비트로 표현
IPv6 주소는 128비트로 표현

주소 체계를 사용하여 데이터가 정확한 호스트로 라우팅되고 목적지를 식별 가능

예) "192.168.0.1"과 같은 IPv4 주소는 해당 네트워크 상에서 특정 호스트를 가리킴
호스트 간 통신은 IP 주소를 사용하여 이뤄지며, 이 주소는 데이터 패킷이 네트워크에서 목적지로 전송되는 데 사용

+ IPv4(IP version 4)와 IPv6(IP version 6)는 인터넷 프로토콜의 버전

### IP 주소는 인터넷에 연결된 컴퓨터 식별자?
각 컴퓨터 또는 네트워크 장치는 고유한 IP 주소를 부여받아 해당 주소를 사용하여 인터넷에서 서로 통신

Port 번호
네트워크 통신에서 프로세스나 서비스를 구분하기 위한 식별자

TCP(Transmission Control Protocol) 또는 UDP(User Datagram Protocol)와 같은 전송 계층 프로토콜에서 사용

한 컴퓨터에서 여러 네트워크 서비스나 프로세스가 동시에 실행될 수 O
각 서비스나 프로세스는 고유한 포트 번호를 갖고 있어서, 해당 번호를 통해 데이터가 어떤 서비스나 프로세스로 전달되어야 하는지 식별

포트 번호는 16비트 숫자, 범위는 0부터 65535까지
이 범위는 0~1023까지는 잘 알려진 포트(Well-Known Ports)로 예약
일반적인 서비스들이 특정 포트를 사용하는데, 예를 들면 HTTP는 80번, HTTPS는 443번을 사용

< 일반적으로 사용되는 포트 범위 >

0~1023: 잘 알려진 포트 (Well-Known Ports)
1024~49151: 등록된 포트 (Registered Ports)
49152~65535: 동적 및 사설 포트 (Dynamic and/or Private Ports)

송신자와 수신자 간의 상호 작용을 통제
데이터가 어떤 프로세스로 전달되어야 하는지 지정하는 데 사용
